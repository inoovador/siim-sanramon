# SIIM F0 — Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Crear el scaffold del nuevo proyecto SIIM (Laravel 11 + Livewire 3 + Clean Architecture) en `siim/`, paralelo al legacy `app/`, con tooling de calidad (Pest, PHPStan max, Pint, Deptrac), paleta brand SIIM aplicada, CI verde, y un smoke test que valide el bootstrap.

**Architecture:** Monolito Laravel 11 + Livewire 3 + Volt en `siim/`. Estructura `app/` (HTTP layer) + `src/{Domain,Application,Infrastructure}` (Clean Architecture, 6 bounded contexts vacíos preparados). Tooling: Pest 3 + PHPStan level max + Pint + Deptrac enforce. Stack runtime: PHP 8.2+ / MariaDB 11 / Redis 7. CI: GitHub Actions con matrix PHP 8.2/8.3.

**Tech Stack:** PHP 8.2, Laravel 11.31, Livewire 3.6 + Volt 1.7, Tailwind 3, Pest 3, PHPStan + Larastan, Pint, Deptrac, MariaDB 11, Redis 7, Docker multi-stage.

**Spec reference:** `docs/superpowers/specs/2026-05-21-siim-design.md` §2, §6, §7, §9.

**No-objetivos F0** (van en fases siguientes):
- Modelos de dominio reales (F1+)
- Migraciones de BD (F2+)
- Login/auth (F1)

## Shell convention (importante)

**Por defecto: usar Git Bash (MINGW64)** para todos los comandos — sintaxis POSIX uniforme.
Cuando un step diga explícitamente `(PowerShell)`, abrir PowerShell para ese step.

Convenciones:
- Paths: usar `/` (forward slash), no `\`.
- Variables: `$VAR` en Bash, `$env:VAR` solo en PS.
- Pipes y filtros: `find`, `grep`, `wc` en Bash. En PS usar `Get-ChildItem`, `Select-String`, `Measure-Object`.
- Comandos `git`: idénticos en ambos shells.

### Task 9: Configurar Deptrac (6 bounded contexts + Clean Arch layers)

**Files:**
- Create: `siim/deptrac.yaml`

- [ ] **Step 9.1:** Crear `siim/deptrac.yaml`

```yaml
deptrac:
  paths:
    - ./src
    - ./app
  exclude_files:
    - '#.*test.*#i'

  layers:
    # Clean Architecture layers (vertical)
    - name: Domain
      collectors:
        - type: classLike
          regex: ^SIIM\\Domain\\.*

    - name: Application
      collectors:
        - type: classLike
          regex: ^SIIM\\Application\\.*

    - name: Infrastructure
      collectors:
        - type: classLike
          regex: ^SIIM\\Infrastructure\\.*

    - name: Http
      collectors:
        - type: classLike
          regex: ^App\\Http\\.*

    # Bounded contexts (horizontal)
    - name: Context_Identity
      collectors:
        - type: classLike
          regex: ^SIIM\\(Domain|Application)\\Identity\\.*

    - name: Context_Citizen
      collectors:
        - type: classLike
          regex: ^SIIM\\(Domain|Application)\\Citizen\\.*

    - name: Context_Ingestion
      collectors:
        - type: classLike
          regex: ^SIIM\\(Domain|Application)\\Ingestion\\.*

    - name: Context_Analysis
      collectors:
        - type: classLike
          regex: ^SIIM\\(Domain|Application)\\Analysis\\.*

    - name: Context_Conversation
      collectors:
        - type: classLike
          regex: ^SIIM\\(Domain|Application)\\Conversation\\.*

    - name: Context_Reporting
      collectors:
        - type: classLike
          regex: ^SIIM\\(Domain|Application)\\Reporting\\.*

  ruleset:
    # Clean Arch deps (lower depends on nothing above)
    Domain: []
    Application:
      - Domain
    Infrastructure:
      - Application
      - Domain
    Http:
      - Application

    # Bounded contexts: cross-context solo via Application/Shared/Contracts (no acoplamiento directo Domain ↔ Domain)
    Context_Identity: []
    Context_Citizen: []
    Context_Ingestion: []
    Context_Analysis: []
    Context_Conversation: []
    Context_Reporting: []
```
